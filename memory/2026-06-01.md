# 2026-06-05 — Transaction Guard Fix

## Bug: Faculty Review Submission Fails

**Symptom:** `POST /api/faculty/questions/:id/review` returned 500 with "Review action failed — please try again" and server log showed:
```
Error: cannot rollback - no transaction is active
```

**Root Cause:** sql.js's `db.export()` (called by `saveDb()`) auto-commits any open BEGIN IMMEDIATE transaction. The `run()` helper calls `saveDb()` after every statement, which was committing the transaction prematurely. So after `db.run('COMMIT')` succeeded, any subsequent throw in the try block would attempt `db.run('ROLLBACK')` on an already-closed transaction.

**Fix — `database.js`:**
- `_inTransaction` flag + `beginTransaction()`/`commitTransaction()`/`rollbackTransaction()` helpers that set it
- `run()` skips `saveDb()` when `_inTransaction` is true

**Files changed:**
- `server/db/database.js` — added guard + 3 helpers, updated `run()`
- `server/routes/api.js` — voteTransaction uses new helpers
- `server/routes/faculty.js` — review, tags/apply, bulk-action routes use new helpers
- `server/utils/migrate.js` — uses new helpers
